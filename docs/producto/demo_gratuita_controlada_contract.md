# Demo Gratuita Controlada

Producto: LexControl  
Instrumento: Activacion comercial inicial  
Version: v1.1 MVP

---

## 1. Decision Congelada

El CTA principal de la landing deja de ser:

```text
Hacer diagnostico
```

Y pasa a ser:

```text
Activar demo gratis
```

El diagnostico no desaparece. Queda como instrumento secundario de conciencia y como insumo de calificacion dentro del flujo de activacion.

---

## 2. Tesis De Conversion

La landing debe llevar al usuario por este flujo:

```text
1. Hay una zona ciega en mi operacion.
2. Esto puede estar pasandome hoy.
3. LexControl me muestra lo que no puedo ver.
4. Puedo probarlo con mis procesos sin comprar todavia.
5. Quiero activar una demo.
```

La demo debe producir esta sensacion:

```text
Esto es lo que necesito ver todos los dias.
```

---

## 3. Nombre Comercial

Nombre publico recomendado:

```text
Demo gratuita controlada
```

CTA principal:

```text
Activar demo gratis
```

CTA alternativo en cierre:

```text
Solicitar activacion
```

Evitar como CTA principal:

- Solicitar beta.
- Acceso beta.
- Hacer diagnostico.
- Agenda una llamada.

---

## 4. A Quien Aplica

La demo esta pensada para:

- Abogados litigantes.
- Firmas pequenas o medianas.
- Operadores de vigilancia judicial.
- Equipos que manejan volumen real de procesos.

Perfil ideal:

```text
50 a 500 procesos activos.
```

No se debe comunicar como herramienta para usuarios casuales con muy pocos procesos.

---

## 5. Que Incluye

La demo gratuita controlada puede incluir:

- Carga inicial de hasta 100 procesos.
- Hasta 4 responsables.
- Bandeja operativa con estados.
- Deteccion de novedades, sin cambios y fallas.
- Registro de consultas.
- Vista de errores de fuente.
- Lex sobre la bandeja.
- Sesion inicial de activacion.
- Revision de resultados con el equipo.

---

## 6. Que No Incluye

La demo no incluye:

- Asesoria juridica.
- Revision juridica de fondo.
- Automatizacion de fuentes con CAPTCHA.
- Garantia de disponibilidad permanente de fuentes externas.
- WhatsApp automatico productivo.
- Carga ilimitada de procesos.
- Acceso automatico sin validacion previa.

---

## 7. Duracion Recomendada

Duracion sugerida:

```text
14 dias.
```

Razon:

- Es claro para el usuario.
- Permite observar novedades reales.
- Mantiene control operativo durante la etapa inicial.
- Evita prometer una beta abierta indefinida.

---

## 8. Condiciones Operativas

La activacion debe ser controlada.

Condiciones:

- El usuario solicita acceso.
- El equipo revisa si la operacion encaja.
- Se coordina una sesion de activacion.
- La carga inicial puede ser asistida.
- La demo se trabaja con procesos reales cuando sea viable.
- No se requieren datos sensibles para solicitar acceso.

Copy recomendado:

```text
Estamos activando un grupo reducido de abogados y firmas que operan volumen real de procesos.
```

No comunicar numero exacto de cupos en esta etapa.

---

## 9. Flujo Del Modal

Decision actualizada:

```text
El flujo detallado de activacion queda definido en:
docs/producto/contrato-flujo-activacion-demo.md
```

Este documento conserva la tesis comercial de la demo.
El contrato especifico define:

- Pasos del flujo.
- Preguntas permitidas.
- Datos que no deben repetirse.
- Criterios de calificacion.
- Salida hacia calendario o revision.

Reglas congeladas:

- Una pantalla pide una sola decision.
- No combinar dos preguntas en un mismo paso.
- No volver a pedir numero de procesos en datos de contacto.
- Solo mostrar calendario si el prospecto califica.

La version anterior de 3 pasos queda reemplazada por el contrato granular de activacion.

---

## 10. Rol Del Diagnostico

El diagnostico queda como:

- CTA secundario.
- Herramienta de conciencia.
- Insumo de calificacion.
- Paso previo posible antes de solicitar activacion.

No debe competir con la demo gratuita como accion principal.

---

## 11. Precios De Lanzamiento

Decision congelada:

```text
La landing debe mostrar precios visibles bajo la etiqueta
Precios de lanzamiento.
```

Razon:

- Reduce la percepcion de que LexControl es inalcanzable.
- Filtra prospectos con intencion real.
- Aumenta confianza antes de solicitar demo.
- Evita que la demo parezca una llamada comercial sin contexto.

La demo gratuita debe aparecer antes de la decision de compra:

```text
Puedes activar una demo gratuita antes de elegir plan.
```

Planes sugeridos:

### Starter

```text
$149.000 COP / mes
Hasta 100 procesos
1 usuario
Bandeja operativa
Consulta CPNU / Rama Judicial
Estados: novedad, sin cambios y fallas
```

### Profesional

```text
$299.000 COP / mes
Hasta 250 procesos
Hasta 2 usuarios
Bandeja operativa
Lex sobre la operacion
Resumen diario
Prioridades
```

### Firma

```text
$599.000 COP / mes
Hasta 500 procesos
Hasta 5 usuarios
Alertas por responsable
Mayor volumen de consulta
Soporte de activacion
```

Para mas de 500 procesos:

```text
Planes Operativo y Enterprise segun volumen.
```

### Complementos Disponibles

En landing se deben mostrar como capacidad adicional, no como precios principales.

Copy publico:

```text
Complementos disponibles.
Activa mas capacidad segun plan y volumen.
```

Complementos visibles:

- WhatsApp operativo.
- Procesos adicionales.
- Reportes automaticos.
- Carga masiva asistida.
- Revision asistida.
- Portal cliente final.

No mostrar precios detallados de complementos en la landing principal.

Rangos internos:

| Add-on | Rango sugerido |
| --- | ---: |
| Proceso adicional | COP $1.000 - $2.500 / proceso / mes |
| Usuario adicional | COP $29.000 - $49.000 / usuario / mes |
| WhatsApp premium | COP $49.000 / mes + consumo |
| Alerta WhatsApp individual | COP $300 - $500 / alerta enviada |
| Reporte mensual automatico | COP $79.000 / cuenta / mes |
| Carga masiva asistida | COP $150.000 - $400.000 unico pago |
| Revision asistida | COP $500 - $1.500 / proceso revisado |
| Portal para cliente final | COP $49.000 - $99.000 / cliente / mes |

No usar:

- Cotizar como CTA principal.
- Plan personalizado como mensaje dominante.
- Precio oculto.

## 13. Criterios De Aceptacion

La activacion esta bien implementada si:

- El CTA principal es `Activar demo gratis`.
- El diagnostico queda como CTA secundario.
- Los precios aparecen como `Precios de lanzamiento`.
- La demo explica que recibe el usuario.
- La demo no promete automatizacion absoluta.
- La solicitud no exige datos sensibles.
- El flujo permite calificar volumen y riesgo.
- El cierre dirige a activacion, no solo a diagnostico.
