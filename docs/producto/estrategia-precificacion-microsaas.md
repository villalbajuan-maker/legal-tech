# Estrategia de Precificacion MicroSaaS

## Contexto

El producto ya no debe pensarse solamente como una automatizacion para Legal Search. La oportunidad es construir un sistema operativo de vigilancia judicial para abogados, firmas pequenas y equipos juridicos.

Horizonte inicial esperado:

- 50 a 80 cuentas de abogados o firmas.
- 3.000 a 4.000 procesos vigilados.
- Consulta automatica prioritaria sobre Rama Judicial / CPNU.
- Expansion gradual a fuentes como SAMAI, SIC, SuperSociedades y SIPI, segun viabilidad tecnica y restricciones de acceso.

La unidad economica principal no debe ser solo el usuario. El valor y el costo operativo se concentran en el proceso vigilado.

## Principio de pricing

El cliente no compra software. Compra control operativo sobre sus procesos judiciales.

El precio debe reflejar:

- Cantidad de procesos vigilados.
- Nivel de automatizacion.
- Trazabilidad y evidencia de consulta.
- Alertas y canales de notificacion.
- Soporte operativo.
- Gestion de riesgo cuando una fuente externa falla o impone restricciones.

Formula comercial recomendada:

```text
Cuenta / firma
+
Volumen de procesos vigilados
+
Nivel de alertas, trazabilidad y soporte
```

## Lectura estrategica

El mercado ya acepta pagar por vigilancia judicial con precio asociado al volumen de procesos.

Hay dos bandas relevantes:

| Categoria | Banda aproximada |
| --- | ---: |
| Automatizacion basica por radicado | COP $1.100 - $1.300 / proceso / mes |
| Servicio profesional asistido | COP $5.700 - $6.000 / proceso / mes |

La posicion recomendada no es competir como el monitor mas barato. La oportunidad es posicionar el producto como un sistema operativo de vigilancia judicial:

```text
Mas control.
Mas trazabilidad.
Mejor bandeja operativa.
Mejor gestion de errores de fuente.
Mejor experiencia para abogados pequenos y firmas medianas.
```

## Posicionamiento recomendado

No vender como:

```text
Monitor automatico de radicados.
```

Vender como:

```text
Sistema operativo de vigilancia judicial para monitorear, priorizar y evidenciar procesos judiciales en multiples fuentes.
```

Promesa comercial:

```text
La plataforma permite a abogados y firmas vigilar procesos judiciales, detectar novedades, controlar actuaciones recientes, recibir alertas y conocer el estado operativo de cada fuente consultada.
```

## Regla de precio por proceso

La regla interna debe ser:

| Nivel | Banda por proceso / mes |
| --- | ---: |
| Automatizado basico | COP $1.100 - $1.500 |
| Operativo con trazabilidad | COP $1.500 - $2.500 |

## Estrategia de entrada

Para los primeros clientes se recomienda un programa fundador:

```text
Primeros 10 clientes beta:
30% de descuento durante 6 meses
a cambio de feedback, casos reales y autorizacion para validar el producto.
```

Condicion importante:

```text
Precio fundador valido solamente durante etapa beta.
```

Esto permite entrar al mercado sin anclar la marca a precios demasiado bajos.

## Consideraciones de riesgo

Las fuentes externas pueden cambiar, limitar consultas, imponer CAPTCHA o modificar sus estructuras.

El precio no debe prometer automatizacion absoluta. Debe prometer control operativo:

```text
Cuando una fuente permite consulta automatica, el sistema la ejecuta periodicamente.
Cuando una fuente impone restricciones tecnicas, el sistema conserva la ultima consulta valida, marca el proceso como pendiente de revision y activa flujos asistidos segun el plan contratado.
```

Esto protege el margen y evita que el producto quede obligado a resolver manualmente todo dentro de planes de bajo costo.
